# Проект "Защита персональных данных клиентов"


## Данные

Признаки: пол, возраст и зарплата застрахованного, количество членов его семьи.
Целевой признак: количество страховых выплат клиенту за последние 5 лет.


## Задача

Требуется защитить данные клиентов страховой компании. 
1. Нужен такой метод преобразования данных, чтобы по ним было сложно восстановить персональную информацию. Обосновать корректность его работы.
2. Нужно защитить данные, чтобы при преобразовании качество моделей машинного обучения не ухудшилось. Подбирать наилучшую модель не требуется.


## Ход исследования
- Подготовили данные, удалили дубликаты, скорректировали типы, посмотрели на распределения - вопросов не возникло
- Доказали через свойства матриц, что качество линейной регрессии при умножении признаков на обратимую матрицу не изменится
- Сформулировали алгоритм кодирования
- Проверили работу алгоритма: 
    - создали матрицу признаков
    - создали случайную обратимую матрицу
    - закодировали матрицу признаков
    - запустили модель линейной регрессии на оригинальной матрице признаков и на закодированной - качество модели да и после преобразования не изменилось
    - чтобы дешифровать данные, умножили зашифрованную матрицу признаков на обратную (матрицу-ключ) справа
    - Проверили, что дешифрованная матрица совпадает с оригинальной


## Выводы
- Для кодирования признаков необходимо сформировать случайную матрицу, умножить ее на матрицу признаков, и предсказания не изменятся, и качество модели не пострадает. 
- При этом мы получили набор случайных векторов, ключом-связкой с исходным набором признаков является обратная к случайной матрица.
